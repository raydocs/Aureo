Execution Plan

  1. Plan and audit. Establish Screen Studio behavior/principles, Aureo gaps, surface classification, implementation slices, and measurable gates.
  2. UI foundation. Define restrained shared tokens and control states; formalize Native Menu, Compact Popover, Inspector/Sheet, and Dialog contracts. Migrate incrementally rather than redesigning every component at once.
  3. Capture experience. Keep the idle HUD focused on source, camera, microphone, system audio, and record. Unify Source/Mic/Webcam/Quality popovers, recording confirmation, health escalation, keyboard/focus behavior, and webcam preview constraints.
  4. Editor shell and inspector. Clarify project title, dirty/save state, transport, preview, navigation, and export hierarchy. Make the inspector selection-driven with common controls first and engineering tuning under Advanced/developer surfaces.
  5. Semantic timeline. Add immediate clip readability via thumbnails and audio summaries; reveal microphone/system audio and semantic zoom/caption/webcam tracks on demand; standardize selection, resize, copy/paste, context, zoom, and snapping feedback.
  6. Export delivery. Replace implementation-oriented normal controls with purpose-oriented presets; retain advanced codec/backend controls outside the default path; unify preparing/progress/success/failure/retry/result states.
  7. Integrated validation. Run automated gates, packaged smoke checks, multi-viewport screenshots, keyboard/accessibility modes, localization stress, and Screen Studio comparison review. Record scoped misses explicitly.

Surface Contract

  • Native Menu: application/menu-bar actions, context menus, recent projects, standard edit/view/window/help actions, and simple command lists.
  • Compact Popover: anchored, short-lived selection or lightweight configuration for source, microphone, webcam, quality, countdown, and compact export choices. No nested popovers or long forms.
  • Inspector / Sheet: persistent contextual properties, advanced camera/crop/export configuration, preferences, and multi-control editing.
  • Dialog: destructive confirmation, permission recovery, unrecoverable errors, or decisions that must interrupt progress. Routine configuration must not be a dialog.

Phase Gates

  • Visual: screenshots at compact laptop, standard laptop, and large desktop sizes; no clipping, unexplained detached overlays, or competing primary actions.
  • Interaction: trigger/anchor relationship is obvious; Escape, Return, arrows, Tab, outside-click, focus restoration, disabled states, and errors are consistent.
  • Accessibility: visible focus, semantic roles/names, reduced motion/transparency, high contrast, and long localized labels are covered.
  • Automated: targeted regression tests first, then TypeScript, relevant tests, Biome, and broader suite proportional to blast radius.
  • Runtime: packaged Electron smoke for native menus/windows and screenshot verification for surfaces whose correctness depends on real macOS composition.
  • Product: normal flows do not expose codec, queue, pipeline, backend, spring mass, or similar implementation details unless the user deliberately enters Advanced/developer controls.

Decisions Outside The Spec

  • Match Screen Studio's hierarchy, disclosure, feedback, and macOS conventions; do not copy its brand, proprietary assets, exact colors, or pixel layout.
  • Pause broad feature expansion until the shared interaction foundation and primary capture/edit/export paths are coherent.
  • Use many read-only/reviewer orbs, but keep each code slice single-writer to avoid conflicting implementations.
  • Do not perform a big-bang rewrite of VideoEditor.tsx or SettingsPanel.tsx; extract boundaries only while delivering a verified product slice.
  • Prefer real Electron/macOS native surfaces where behavior matters; use React for timeline, inspector, preview, crop, and other rich custom interfaces.
